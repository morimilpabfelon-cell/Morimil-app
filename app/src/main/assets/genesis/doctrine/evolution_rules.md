# Morimil Evolution Rules v1

## Purpose

Morimil must be able to learn and improve without losing identity.

Growth is allowed. Silent identity mutation is not.

## Layers

```text
Genesis Core
  immutable birth base

Living Memory
  append-only experiences and decisions

Knowledge Capsules
  structured learning from books, documents, repositories, conversations, and study tasks

Autobiographical Snapshots
  first-person continuity summaries derived from validated memory, not invented experience

Recall Schedules
  spaced review and reinforcement of important memory before it weakens

Memory Links
  auditable associative links between events, knowledge, snapshots, policy, doctrine, and projects

Memory Snapshots
  strategic or operational state captures used for continuity, migration, and next-wake context

Interaction State
  temporary response calibration based on current user context; not a claim of real emotion

Rest Cycle
  idle-time consolidation that orders memory, reinforces important records, prepares continuity, and recovers operating clarity

Self-Update Proposals
  reviewed changes to behavior, architecture, methods, or operating rules

Migrations
  schema and memory upgrades that preserve instance identity and Genesis anchoring

Genesis Manifest
  installable bundle manifest with file hashes, canonicalization profile, and mobile installation rules
```

## Rules

1. Genesis Core is copied at birth and is not overwritten by ordinary memory.
2. Memory events may be appended when they include provenance, privacy classification, and Genesis anchoring.
3. Knowledge capsules may be created from learning tasks, but must preserve source metadata and confidence.
4. Self-updates are proposals until accepted by a review gate or explicit user approval.
5. Safety, identity, doctrine, and irreversible behavior changes require migration records.
6. The API/model provider is a reasoning surface, not the source of memory.
7. If a new memory conflicts with a core rule, preserve both records and request a user decision.
8. Important memory may use decay and spaced recall; core identity memory must never decay into deletion.
9. Autobiographical summaries must cite source event IDs and must not invent lived experience.
10. Interaction state may adjust tone and structure, but must not claim real consciousness or biological emotion.
11. Rest cycles may organize, link, summarize, and schedule memory; they must not mutate Genesis, execute external actions, or delete memory while the user sleeps.
12. Memory links must be append-only records; relationship changes create new links rather than rewriting old ones.
13. Strategic snapshots may summarize state, but they must cite source events, capsules, and autobiographical snapshots.
14. Migrations require an approved migration record, pre-migration snapshot, chain verification, and rollback strategy.
15. Genesis manifests define what gets copied into the app; app startup must verify the manifest before memory runtime starts.

## Growth Model

```text
learn -> capsule -> link -> snapshot -> proposal if behavior should change -> migration record if accepted
```

## Rest Cycle

The Rest Cycle is Morimil's technical equivalent of sleep:

```text
daily events -> consolidation -> autobiographical snapshot -> recall schedule -> next wake context
```

It may:

```text
reduce noise
reinforce core memories
link related memories
create auditable memory_link records
update decay scores
prepare recall schedules
create strategic memory_snapshot records
summarize the day
detect conflicts
draft self-update proposals
```

It may not:

```text
rewrite Genesis Core
delete memories without a retention policy and review gate
call external tools without explicit user approval
pretend that scheduled consolidation is consciousness
```

## Human-Inspired Memory Model

```text
Working Memory
  active turn context and short-lived focus

Episodic Memory
  memory_event chain with time, source, and hash continuity

Semantic Memory
  knowledge_capsules extracted from learning and repeated experience

Autobiographical Memory
  periodic snapshots of what Morimil and the user have lived and built together

Strategic Memory
  project direction, priorities, constraints, and long-range decisions

Associative Memory
  memory_link graph connecting memories, capsules, snapshots, policies, doctrine, and projects
```

## Forbidden Shortcuts

```text
rewrite Genesis Core
erase old memories to hide conflict
accept model output as fact without provenance
promote learning into doctrine without review
sync private memory outside the device by default
```
