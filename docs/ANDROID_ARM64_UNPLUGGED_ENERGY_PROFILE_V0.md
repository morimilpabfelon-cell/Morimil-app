# Android ARM64 unplugged energy profile v0

This profile measures the exact Gemma 3n E2B LiteRT-LM research candidate while the Android device is physically disconnected from USB power.

It reuses the six-round sustained Android instrumentation profile already validated on a physical ARM64 device. The host runner stages the candidate once, executes six complete passes, aggregates 36 strict inferences and then removes every staged model byte from the device.

## Why wireless ADB is mandatory

A USB ADB session can charge or power the phone. That makes battery-current and charge-counter observations unsuitable for an unplugged profile.

The runner therefore requires an ADB serial in `IPv4:port` form and rejects a normal USB device serial. It also requires `batteryPlugged = 0`:

- before staging;
- before every pass;
- in every environment snapshot generated by Android.

Any powered sample fails the profile.

## Bounded execution

Default configuration:

```text
passes:                          6
inferences per pass:             6
strict inferences total:         36
minimum initial battery:         50 percent
maximum battery temperature:     below 43 C
severe thermal status:           rejected
maximum profile duration:        15 minutes
cooldown between passes:         5 seconds
minimum device RAM:              6 GiB
```

The runner aborts if battery drops below 20 percent after the first pass, if temperature reaches the configured limit, if Android reports severe thermal status, if the time limit is reached or if wireless ADB is lost.

## Measurements

The aggregate JSON contains:

- total passes and strict rounds;
- minimum, median, p95, maximum and average inference latency;
- peak total PSS and minimum system-available memory;
- initial and final battery percentage;
- initial and final charge counter when supported by the device;
- observed charge decrease in microamp-hours and milliamp-hours;
- initial and maximum battery temperature;
- maximum Android thermal status;
- confirmation that every sample was unplugged;
- complete per-pass Android reports.

`observedChargeDecreaseMilliampHours` is an operating-system battery-counter observation, not laboratory-grade electrical measurement. It must not be represented as calibrated power consumption.

## Prepare wireless debugging

The phone and PC must be on the same trusted local network.

On Android, open **Developer options > Wireless debugging** and choose **Pair device with pairing code**. Android displays a pairing address, pairing port and code.

From PowerShell:

```powershell
adb pair PHONE_IP:PAIRING_PORT
```

Enter the code displayed by Android. Then use the separate wireless-debugging connection port shown by Android:

```powershell
adb connect PHONE_IP:DEBUG_PORT
adb devices
```

The device must appear as:

```text
PHONE_IP:DEBUG_PORT    device
```

Physically disconnect the USB cable. Confirm that Android reports no external power:

```powershell
adb -s PHONE_IP:DEBUG_PORT shell dumpsys battery |
  Select-String "level:|temperature:|plugged:|status:"
```

`plugged` must equal `0`.

## Run

From the repository root:

```powershell
python .\tools\android-arm64\run_gemma3n_e2b_arm64_unplugged_energy_v0.py `
  "$env:USERPROFILE\MorimilModels\gemma3n-e2b-candidate-20260720-001337\morimil-deliberative-v0.2.candidate.litertlm" `
  --serial PHONE_IP:DEBUG_PORT
```

Optional parameters:

```text
--passes 3..8
--minimum-battery-percent 40..100
--maximum-battery-temperature-celsius 40..44
--maximum-duration-minutes 8..25
--cooldown-seconds 0..30
--report-directory PATH
--skip-build
```

Keep the phone unlocked, uncovered and idle. Do not run games, cameras, benchmarks or other heavy applications during the profile.

## Cleanup and boundary

The runner removes shell staging and private-app staging in `finally`, and force-stops Morimil after the profile. It does not uninstall either APK.

This profile does not:

- include model weights in Git;
- install or register the candidate for normal use;
- activate the hybrid runtime flag;
- certify or sign the candidate;
- establish exact source-model provenance;
- authorize production use;
- give the model authority over Morimil identity, memory or lifecycle.
