# Morimil App Phase 5F Status

Phase 5F is now provider-neutral.

Memory: phone-local Room/SQLite. It grows through local events and remains the source of continuity.

Reasoning motor: stateless transport between calls. The active slot supplies endpoint, model, wire format, and encrypted runtime key.

Runtime screen: supports up to 10 API slots, model discovery from compatible catalogs, manual endpoint/model override, and active-slot selection.

No provider owns identity, memory, or doctrine. The API only reasons over the local context Morimil sends for that turn.
